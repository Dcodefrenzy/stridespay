Gulp tasks

Dashield comes with multiple gulp tasks that will simplify building you application. To be able to use gulp, you must access your command prompt and cd to your project folder, e.g cd C:\Users\YOUR_DIRECTORY\Documents\Dashield. This will scope the command line to the directory of your choice. Then you can run any of the following tasks.

- gulp inject Injects partials from the partial folder into the respective positions of the page.

                  
                    // Your code here

This way, anything that lives in _globalStyles.html partial will be injected between the two lines.

- gulp vendorScripts Bundles all scripts assigned in the task which can be found in ./gulpfile.js into 1 single js file. (By default this is being placed inside vendor/dist/libs.bundle.js )

- gulp vendorStyles Bundles all style sheets assigned in the task which can be found in ./gulpfile.js into 1 single js file. (By default this is being placed inside vendor/dist/libs.bundle.css )

- gulp styles Minifies the main Dashield stylesheets and places it in vendor/dist/style.bundle.css

- gulp scripts Minifies the main Dashield Scripts and places it in vendor/dist/script.bundle.js

3rd Party Libraries

Dashield bundles all libraries (with a few exceptions) into 1 Javascript file, and 1 Css file. However, you can always use gulp vendorScripts and gulp vendorStyles to remove any unused script or stylesheet from the bundles

Some library intializations are available right off the bat, and only needs a utility class to start using them. The following utility classes can be used to activate libraries:

- .dsh-select2 Converts any list into a select2 dropdown list

- .dsh-datepicker Converts any input field into a date picker

- .dsh-timepicker Converts any input field into a time picker

- .dsh-daterangepicker Converts input field into a date range picker

- .dsh-daterangepicker-time Converts any input field into a date range picker with the ability to also filter time

- .dsh-daterangepicker-predefined Converts any input field into a dropdown with predefined dates, such as today, last 7 days, etc...

- .dsh-tooltip or data-toggle="tooltip" Adds a tooltip to any element

This utility has multiple data attributes that you can use to show your tooltips in a different way:

  • data-state which accepts primary, secondary, success, warning, danger, dark, info, or light
  • data-trigger which accepts click, focus or hover
  • data-placement which accepts top, left, right, or bottom
  • title The text which will be displayed in the tooltip

- .dsh-popover or data-toggle="popover" Adds a popover to any element

All attributes available for popovers are similar to those of tooltips

- .needs-validation and novalidate are a class and attribute respectively which should be added to any form that required client-side validation. Simply add those 2 to any form element and you will implement bootstrap validation

- .dsh-custom-scroll adds a clean scroll bar to an element. Note that this element must have a predefined height

Everything else in terms of libraries, should be initialized manually. However, don't worry. Each library has a set of examples which you can find in assets/js/examples/~

Select / Deselect All select2 multi tags

To be able to add the Select all option and Deselect All option, you can use the following classes:

- .dsh-select-all and .dsh-deselect-all

It is important to wrap the whole thing in a form-group class, otherwise those classes will not work.

Copy / Cut to clipboard

- .dsh-clipboard to any element or input to copy to clipboard. Add the class .cut to cut the content to clipboard instead

Activate rating

Dashield allows you to have your rating enabled or disabled by simply adding 1 class:

- .dsh-can-rate can be used around rating elements to allow them to be hoverable and clickable. Clicking an child of .dsh-can-rate will give it the active class

Data Tables

As mentioned above, not all 3rd party libraries are included in the libs.bundle.js file. One of the scripts would be datatables.min.js. This is due to how big the file is when you include the export plugins.

If you are not going to use the export plugins PDF export, CSV export, Print, or HTML export, we recommend you head over to datatables main page, and redownload the file with only the core datatables js, excluding the bundleded plugins that we have in our version of datatables.

Now that this is taken care of, let's go over this section.

We have bundleded all plugins with datatables, and provided an example for each one. Each page's example can be found in the help popup which can be accessed by clicking the button at the bottom right of your screen.

Charts

Similar to datatables, apex charts is not included in bundle due to it being a large file (approx. 1MB), and it should only be used on demand

Google Fonts

By default, Dashield used Poppins as the main font accross the whole template. You can change that by accessing the page and changing the Font URL at the very start of the head tag. Simply go to Google fonts, select the font you like, and replace the link with the one we currently have accross all our pages.

Quick tip would be to search and replace all occurences. To begin, copy the Above line and replace it in your project. This is different based on the IDE being used
